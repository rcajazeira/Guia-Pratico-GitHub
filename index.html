Guia Completo: Git e GitHub

1. Criar Novo Repositório no GitHub

  1. Acesse github.com
  2. Clique no ícone '+' no canto superior direito
  3. Selecione "New Repository"
  4. Configure o repositório:
    • Nome do repositório (use hífens ou underscores, sem espaços)
    • Descrição (opcional)
    • Visibilidade: Public ou Private
    • Não inicialize com README, .gitignore ou licença (vamos fazer isso manualmente)
  5. Clique em "Create repository"

2. Configuração Inicial do Git (primeira vez)

git config --global user.name "Seu Nome"
git config --global user.email "seu@email.com"

3. Preparar Projeto Local

cd caminho/para/onde/quer/salvar/projeto
git clone https://github.com/seu-usuario/nome-do-projeto
cd "nome-do-projeto"

4. Inicializar e Configurar Repositório

git init
git remote add origin https://github.com/seu-usuario/nome-do-projeto
git branch -M main
git status
git add .
git commit -m "Adicionando arquivos iniciais do projeto"
git push -u origin main

5. Comandos Úteis do Dia a Dia

git status # Verificar estado dos arquivos
git log # Ver histórico de commits
git pull origin main # Atualizar repositório local
git branch # Listar branches
git checkout -b nova-branch # Criar e mudar para nova branch
git checkout nome-da-branch # Mudar de branch

6. Configurar GitHub Pages

  1. No repositório do GitHub, vá para "Settings"
  2. No menu lateral, clique em "Pages"
  3. Em "Build and deployment":
    • Source: "Deploy from a branch"
    • Branch: "main"
    • Pasta: "/root"
  4. Clique em "Save"
Dicas importantes:
  • Certifique-se que seu arquivo index.html está na raiz do projeto
  • Use caminhos relativos para links e recursos
  • GitHub Pages está disponível gratuitamente apenas para repositórios públicos
  • O site pode demorar alguns minutos para ficar disponível
  • O link será: https://seu-usuario.github.io/nome-do-repositorio/
  • É possível configurar domínios personalizados
Solução de Problemas Comuns:
  • Se encontrar erro de "index.lock", use: rm .git/index.lock
  • Se der erro no push, verifique se está na branch correta
  • Sempre faça pull antes de começar a trabalhar em um projeto compartilhado
